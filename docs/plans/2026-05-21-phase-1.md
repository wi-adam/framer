# Phase 1 Plan: Straight-Wall Framing Slice

Phase 1 should prove the product loop end to end: author intent, generate a
framing plan, inspect the result, and get a BOM.

## Current Alpha Checklist

Status: Phase 1 is complete and preserved as the `Wall Demo` workflow. The
current app default has moved beyond this slice to the multi-wall shell alpha
described in [../vision.md](../vision.md) and
[../architecture.md](../architecture.md).

- [x] Open or create a schema-versioned `.framer` JSON project.
- [x] Preserve authored wall and opening intent as the project source of truth.
- [x] Edit wall dimensions, stud spacing, opening kind, center, width, height,
  and sill height parametrically.
- [x] Place door, window, and garage-door-style openings from the app catalog.
- [x] Generate deterministic straight-wall framing members from authored intent.
- [x] Inspect generated framing in elevation and a first 3D workspace direction.
- [x] Select generated members and inspect source object plus rule provenance.
- [x] Show starter-profile and unsupported-condition diagnostics.
- [x] Produce a grouped BOM/cut list.
- [x] Export a drawing-oriented SVG artifact and BOM-oriented CSV artifact.
- [x] Save, reopen, regenerate, and verify deterministic project round trips in
  tests.

## Workstream A: Domain Model

Define a compact, testable model for:

- Units and dimensions.
- Walls and wall openings.
- Code/material profile defaults.
- Validation diagnostics for impossible geometry.

Done when a wall with doors/windows can be serialized, validated, and passed to a
solver without GUI knowledge.

## Workstream B: Framing Solver

Generate a deterministic straight-wall framing plan:

- Bottom and top plates.
- Common studs at configured spacing.
- King studs, jack studs, headers, rough sills, and cripple studs at openings.
- Grouped BOM by member use, profile, cut length, and quantity.

Done when solver tests cover wall-only, door, window, invalid opening, and BOM
grouping scenarios.

## Workstream C: Desktop Shell

Create a native Rust app for the first editing loop:

- Wall dimension controls.
- Opening controls.
- Framing elevation viewport.
- BOM panel.
- Clear validation feedback.

Done when `cargo run -p framer-app` opens a usable single-wall editor.

## Workstream D: Project Files

Add project save/load once the first model shape settles:

- JSON project format with schema version.
- Round-trip tests.
- File menu entries.

Done when a demo wall can be edited, saved, reopened, and regenerated with the
same plan.

## Workstream E: Code-Rule Expansion

Turn the starter profile into explicit rule data:

- Header sizing inputs.
- Wall type assumptions.
- Species/grade/load placeholders.
- Unsupported-condition warnings.

Done when the app can explain which assumptions produced a generated member.

Current slice: generated members carry starter rule provenance and the app shows
diagnostics for incomplete code/profile coverage. Full IRC tables, local
amendments, and engineered garage-door/header sizing remain future work.

## Non-Goals For Phase 1

- Complete IRC compliance.
- Multi-story structural analysis.
- Roof or floor framing.
- Freeform mesh editing.
- Construction drawings suitable for permit submission.
